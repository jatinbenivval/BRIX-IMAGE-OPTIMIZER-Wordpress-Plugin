## 2. Security & Data Integrity đź›ˇď¸Ź
All code must adhere to the **"Sanitize early, Escape late"** principle.

### A. Data Sanitization (Input)
Any data coming from `$_POST`, `$_GET`, or `$_REQUEST` must be sanitized before use.
- Use `sanitize_text_field()` for strings.
- Use `absint()` or `intval()` for integers.
- Use `sanitize_email()` for emails.

### B. Data Escaping (Output)
Any data being echoed to the screen or inserted into attributes must be escaped.
- Use `esc_html()` for plain text.
- Use `esc_attr()` for HTML attributes.
- Use `esc_url()` for URLs and `src` attributes.
- Use `esc_js()` for data inside JavaScript blocks.

### C. Nonces & Permissions
- **Nonces**: Every AJAX call or Form submission must verify a nonce using `check_ajax_referer()` or `wp_verify_nonce()`.
- **Capabilities**: Always check user permissions using `current_user_can('manage_options')` before executing sensitive logic.

## 3. Metadata & Repository Standards đź“„
### A. Plugin Header (`plugin-name.php`)
Must include:
- `Plugin Name`, `Version`, `Author`, `License: GPLv2 or later`.
- `Requires at least`: (Minimum WP version).
- `Tested up to`: (Latest WP version).
- `Requires PHP`: (Minimum PHP version, 7.4+ recommended).

### B. Readme File (`readme.txt`)
- Must be named **`readme.txt`** (not .md for the WP repository).
- Must follow the [WordPress Readme Standard](https://wordpress.org/plugins/developers/readme-validator/).
- Includes: `Contributors`, `Tags`, `Stable tag`, `Description`, `Installation`, `FAQ`, `Changelog`.

## 4. Licensing & Legal âš–ď¸Ź
- **License**: Must be **GPLv2 or later** or compatible.
- **Credits**: External libraries must also be GPL-compatible and properly credited.
- **No Tracking**: No telemetry, phone-home, or data collection without explicit user opt-in.
- **No Hidden Links**: No "Powered by" links on the frontend without permission.

## 5. Clean Lifecycle Management đź”„
- **Deactivation**: Should stop functionality but preserve user settings (unless small/temporary).
- **Uninstallation**: Must include an `uninstall.php` file to delete all custom database options, tables, and files generated by the plugin. Use `WP_UNINSTALL_PLUGIN` constant for security.
